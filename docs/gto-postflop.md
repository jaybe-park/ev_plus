# 포스트플랍 GTO 추상화 계획

포스트플랍 GTO 데이터를 어떻게 확보할지에 대한 설계 문서. (2026-07-10 확정)

---

## 문제: 조합 폭발

프리플랍은 169핸드 × 상황 ~200개로 전부 수집이 가능했지만, 포스트플랍은 자릿수가 다르다.

| 구분 | 규모 |
|---|---|
| 플랍 텍스처 (수트 대칭 제거) | 1,755가지 |
| 정확한 플랍 보드 × 홀카드 | ~128만 |
| 턴/리버 + 벳사이즈 트리까지 | 수백만+ |

**"전부 수집"은 원천적으로 불가능하다.**

참고: GTO Wizard 자신도 포스트플랍을 전부 사전 계산해두지 않는다 —
특정 보드를 요청하면 그 순간 CFR 솔버를 돌리는 **온디맨드 방식**이다.
걔네는 "무제한 실시간 계산 능력"으로 조합 폭발을 풀고, 우리는 그 능력이
없으므로(무료 한도 100스팟/일) 다른 방식이 필요하다.

---

## 결정된 전략: 추상화 + 실전 우선순위 + equity 폴백

포커 AI 연구(Libratus, Pluribus)의 표준 기법인 **카드 추상화(card abstraction)**의
축소판. 세 요소가 하나의 시스템으로 맞물린다:

```
① 추상화(스팟 키 정의)  — "무엇을 하나의 스팟으로 칠 것인가"
   정확한 카드가 아니라 텍스처 클래스로 키를 정의
   → 조합 공간이 수백만 → 수백 개로 압축, UNIQUE saturate 가능해짐

② 실전 우선순위 큐     — 그 정의대로 실제 만난 클래스부터 기록
   gto_missing_spots의 프리플랍 방식을 클래스 키로 확장

③ equity 폴백          — 아직 수집 안 된 클래스는 기존 equity 휴리스틱
   (현행 봇 로직 그대로 = 이미 구현돼 있음, 비용 0)
```

⚠️ ①이 ②의 전제조건이다. 정확한 카드로 키를 잡으면 실전에서 같은 보드를
두 번 만날 일이 없어 큐가 무한정 자란다 (saturate 실패).

## 스팟 키(클래스) 정의 — v1

```
(포지션관계, 팟타입, 보드텍스처클래스)
```

- **포지션관계**: IP / OOP (2)
- **팟타입**: SRP(싱글레이즈팟) / 3BP(3벳팟) (2)
- **보드 텍스처 클래스** (플랍 3장 기준):
  - 페어드 여부: 무페어 / 페어드 (2)
  - 수트: 레인보우 / 투톤 / 모노톤 (3)
  - 하이카드: A하이 / K·Q하이 / J·T하이 / 로우 (4)
  - 연결성: 커넥티드(스트레이트 코디네이션) / 드라이 (2)
  - → 텍스처 약 48클래스 (불가능 조합 제외하면 그 이하)

총 클래스 수: 2 × 2 × 48 ≈ **최대 ~192개** — 프리플랍과 같은 자릿수로 압축됨.
핸드(169)는 키에 넣지 않는다 — GTO Wizard가 스팟당 전체 핸드 매트릭스를
주므로 프리플랍과 동일하게 데이터 쪽에 포함된다.

**v1 범위는 플랍만.** 턴/리버는 클래스를 곱하면 다시 폭발하므로 equity 폴백 유지.
액션 라인(체크-벳, 벳-레이즈 등)도 v1에서는 "첫 액션(C벳 상황)"만 — 라인 확장은 v2.

## 수집: 대표 보드 샘플링

GTO Wizard는 클래스가 아니라 **구체적 보드 하나**에 대해서만 답을 준다.
따라서:

1. 클래스마다 **대표 보드 2~3개**를 뽑는다 (클래스 내 무작위 or 전형적 보드)
2. 각 대표 보드의 정답을 GTO Wizard에서 수집 (Playwright 자동화 재사용)
3. 2~3개의 빈도를 평균(블렌드)해서 그 클래스의 전략으로 저장
4. 봇/힌트는 실전 보드 → 클래스 분류 → 클래스 전략 조회

근사 오차 트레이드오프: 클래스를 세밀하게 나눌수록 정확하지만 수집량 증가.
v1은 위 분류로 시작하고, 아레나 A/B로 "equity 휴리스틱 대비 개선"이
확인되는 클래스만 세분화한다.

**목표 설정 (정직하게)**: 완전한 GTO가 아니라 **"equity 휴리스틱보다 확실히
나은 근사"**. 검증 기준은 아레나 bb/100이다 — 개선이 안 나오면 되돌린다.

## 저장 스키마

기존 `gto_postflop_situations` / `gto_postflop_hands` 테이블(v2에 선반영,
현재 미사용)을 활용한다. `action_sequence`에 텍스처 클래스 키를 넣거나
컬럼을 조정한다 (구현 시 판단). 미수집 큐는 `gto_missing_spots`를
street='flop' + 클래스 키로 확장.
